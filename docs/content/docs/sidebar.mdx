The sidebar is the main way you navigate Controller. From top to bottom:

## Projects

The **Projects** list holds every local directory you've added. Each project
expands and collapses so you can focus on the one you're working in. Use
**Add a project** to register a new directory — see [Your first
project](/docs/first-project).

## Sessions

Under each project sit its **Sessions** — one entry per conversation with an
agent. Status indicators show which sessions are running, idle, or finished, so
you can tell at a glance where an agent is busy. Click any session to resume it
with its full history intact.

## Schedules

When a project has scheduled sessions, a **Schedules** list appears for it. Each
schedule starts a new session on its target worktree when it fires. See
[Worktrees](/docs/worktrees) for targeting a worktree, and the schedule CLI for
creating them.

## Settings

The **Settings** button opens the configuration dialog, where you set provider
**API keys** and **model defaults**. Keys are stored by Controller and passed to
the agent process as environment variables — you don't paste them into prompts.

## Help

The **Help** icon opens this documentation site in your browser.

## On mobile

On a narrow screen the sidebar collapses into a slide-over **Sheet** so the chat
gets the full width; tap to open it, and it dismisses when you make a selection.
This mirrors the responsive primitives used across the app. To use Controller
from a phone in the first place, see [Remote access](/docs/remote-access).
