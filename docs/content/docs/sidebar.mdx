The sidebar is the main way you navigate Controller. From top to bottom:

## Radar and Controller Mode

The top of the sidebar is your **radar** — the sessions you're actively keeping
an eye on, each showing its provider icon and a spinner while the agent is
working. Alongside it sits the **Controller Mode** toggle, which turns on the
always-on overlay for steering long runs. See [Controller
Mode](/docs/controller-mode).

## Projects

The **Projects** list holds every local directory you've added. Each project
expands and collapses, and a **New** button adds another. Per-project controls
let you create a worktree, edit, or delete the project. Use **Add a project**
to register a new directory — see [Your first project](/docs/first-project).

## Worktrees

Expand a project to see its **worktrees** — the isolated git checkouts it
manages. Each project starts with its main worktree; the per-project **New
worktree** button adds more. See [Worktrees](/docs/worktrees).

## Sessions

Expand a worktree to see its **sessions** — one entry per conversation with an
agent, each with a provider icon and a status indicator (a spinner shows while
the agent is working). Click any session to resume it with its full history
intact. Long lists load in batches with a **Show more** control.

## Settings

The **Settings** button in the footer opens the configuration dialog, where you
set provider **API keys**, **model defaults**, and manage your **unified
skills**. Keys are stored by Controller and passed to the agent process as
environment variables — you don't paste them into prompts. See
[Skills](/docs/skills) for the skills catalog.

## Help

The **Help** icon in the footer opens this documentation site in your browser.

## On mobile

On a narrow screen the sidebar collapses into a slide-over **Sheet** so the chat
gets the full width; tap to open it, and it dismisses when you make a selection.
This mirrors the responsive primitives used across the app. To use Controller
from a phone in the first place, see [Remote access](/docs/remote-access).
