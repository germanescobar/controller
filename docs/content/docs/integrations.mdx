title: Integrations
description: A uniform gateway over MCP servers, REST APIs, and native CLIs — without pasting secrets.
---

The **integrations gateway** gives the agent a single, uniform surface over
third-party services, no matter how each one is actually reached: MCP servers,
REST APIs, or native command-line tools all look the same from the agent's
side.

## Built-in integrations

Controller ships with integrations for:

| Integration | Kind |
|---|---|
| Cloudflare | MCP server |
| Help Scout Docs | REST API |
| Notion | Native CLI (`ntn`) |
| Trello | REST API |
| Interactive Brokers | Local gateway |
| GitHub Issues | Native CLI (`gh`) |
| Tavily | Web search / extraction |

## Credentials stay out of the prompt

Controller **injects credentials** into each integration call. You never paste
secrets into the app or into the agent's prompt, and the agent never sees the
raw tokens — it just calls the tool and Controller supplies the auth.

## How you use them

You don't call integrations yourself — the **agent** does, on your behalf,
through the `controller-integrations` skill. When a task needs an external
service, the agent searches the gateway for a matching capability and invokes
it, with Controller supplying the credentials.

Just ask in plain language for something that involves a connected service —
"open a Trello card for this bug" or "search Notion for the launch doc" — and
the agent routes through the gateway automatically.
