---
title: Controller Mode
description: The always-on overlay for steering long agent runs with keyboard shortcuts.
---

**Controller Mode** (formerly *Focus Mode*) is an always-on overlay designed
for long agent runs. Instead of watching a single chat scroll by, it keeps the
controls you need to steer the agent within reach while it works.

## When to use it

- **Use Controller Mode** for long, multi-step runs where you want to advance,
  stay, or redirect the agent without hunting through the UI — keyboard-first
  steering over an extended session.
- **Use plain chat mode** for quick, conversational back-and-forth where you're
  reading and replying to each response as it lands.

## Keyboard shortcuts

Controller Mode is modifier-based and **user-customizable**. Shortcuts are
chords built on a modifier (**Ctrl** on Linux, **⌘** on macOS) plus a key, so
they don't collide with the agent's own typing. You can remap them to keys that
suit your keyboard.

## The stay / continue chord

When the agent reaches a point where it's about to advance focus, Controller
shows a **focus-advance toast**. The toast displays the live **stay /
continue** chord so you can decide in the moment:

- **Stay** — hold focus where it is.
- **Continue** — let the agent advance.

The chord shown on the toast is the live, current binding, so it always matches
whatever you've configured — no need to remember the mapping.
